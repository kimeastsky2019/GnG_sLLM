# GnG AI Energy sLLM 플랫폼

이 프로젝트는 **에너지 관리 및 효율화**를 위한 **AI 기반 sLLM(Small Language Model) 솔루션**입니다.
에너지 데이터 분석, 지능형 모니터링, 그리고 sLLM을 활용한 자연어 기반의 에너지 인사이트를 제공하여 에너지 분야의 디지털 전환을 가속화합니다.

# 기술 스택

이 프로젝트는 다음 기술 스택을 사용합니다:
- Vite
- TypeScript
- React
- shadcn-ui
- Tailwind CSS


# 개발 프로세스

1. 사용자 요구사항을 참고하여 `src/index.css`와 `tailwind.config.ts`의 테마 스타일을 조정합니다.
2. 사용자 요구사항에 따라 구현해야 할 페이지를 구분합니다.
3. 각 페이지에서 구현해야 할 기능을 정리하고, `pages` 아래에 해당 폴더와 진입점 파일 `Index.tsx`를 생성합니다.
4. `App.tsx`에서 라우팅 구성을 생성하고, 앞서 만든 각 진입점 파일 `Index.tsx`를 가져옵니다.
5. 정리된 요구사항에 따라, 요구사항이 간단하다면 `Index.tsx`에서 해당 페이지의 모든 작업을 직접 완료할 수 있습니다.
6. 요구사항이 복잡한 경우, 페이지를 여러 컴포넌트로 나누어 구현하며 디렉토리 구조는 다음과 같습니다:
    - `Index.tsx`: 진입점
    - `/components/`: 컴포넌트
    - `/hooks/`: 훅 (Hooks)
    - `/stores/`: 복잡한 상호작용 통신이 필요한 경우 `zustand`를 사용하여 통신합니다.
7. 요구사항 완료 후, `pnpm i`로 의존성을 설치하고 `npm run lint & npx tsc --noEmit -p tsconfig.app.json --strict`를 실행하여 검사 및 문제 해결을 수행해야 합니다.

# 백엔드 인터페이스 연동
- 인터페이스를 새로 추가하거나 Supabase를 조작해야 할 때, 먼저 `src/api`에 해당 API 파일을 추가하고 대응하는 데이터 타입을 내보내야 합니다. `src/demo.ts` 파일을 참고할 수 있으며, Supabase인 경우 구현도 잘 마무리해야 합니다.
- 프론트엔드와 Supabase 구현 시, 정해진 데이터 타입에 따라 완벽하게 구현해야 하며, 가능한 한 이미 정의된 데이터 타입을 수정하지 않도록 주의해야 합니다. 수정이 발생하는 경우, 해당 타입을 참조하는 모든 파일을 확인해야 합니다.
